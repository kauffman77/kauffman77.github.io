Coding Environment Setup

Students are free to use whatever coding environment they wish to get code onto Zaratan for compilation and runs.

Those in search of guidance may wish to set up their coding environment identically to current CMSC216 students by following the posted guide for them:

  • CMSC216 Setup Guide (Google Doc)
  • CMSC216 Setup Walkthrough (Youtube)

This setup favors the following tools

  • VSCodium for Editing
  • SFTP Extension for transferring files back and forth

The setup also covers how to configure VPN+SSH Keys to get password-free login to Zaratan which is HIGHLY RECOMMENDED for all students to consider.

Special Notes on Use of VSCode vs VSCodium

VSCode and VSCodium have the same core implementation and features but diverge in the following:

  • VSCodium is open source, unbranded, and is free of AI tools by default
  • VSCode is Microsoft branded and has many AI-based features enabled by default which are difficult to disable

Students are advised NOT to use editors like VSCode which have AI Features enabled. Any use of AI during exams is against syllabus policies and will result in students being submitted to the Office of Student Conduct for academic integrity breeches. Several students in past classes have suffered this though they claimed they only "accidentally" used AI. Using an AI-free editor like VSCodium, VIM, or Emacs will ensure no 'accidents' happen. It will also help students to focus on learning to code themselves rather than offloading this to others.

Special Notes on use of Remote Editing Extension for VSCode/VSCodium

Students are discouraged from using the popular "Remote Editing" extension for VSCode/VSCodium when connecting to Zaratan. System admins report historically that this extension tends to create lots of processes which do disappear after login and may result in a user being locked out of Zaratan.

For code transfer, favor simple methods like the SFTP extension, or if your system supports it, a remote mount via the SSH Filesystem (Linux). Both these methods appear to play nicely with Zaratan.

For interacting on Zaratan, favor using plain SSH to get onto the system in a terminal.

More information on the difficulties with the VSCode Remote Editing extension problems on Zaratan are here: https://hpcc.umd.edu/faq/connection/#why-was-i-advised-not-to-use-vscode-on-the-hpc-systems
