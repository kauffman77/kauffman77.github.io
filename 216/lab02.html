1 Rationale

This exercise demonstrates a number of C programming conventions through a small application. Studying the code, correcting some mistakes in it, and adding some new functionality will familiarize students with the basic edit/compile/test cycle common to coding projects. Completing the exercise will also show students a common application structure for C programs, dividing code between a main routine and a one or more "service" libraries that manipulate data structures. This approach will be used in an upcoming project.

Note: The techniques and syntax used in the provided code will eventually be described in lecture although some will not be discussed until after the exercise is due. This should encourage students to study the code carefully, experiment with it, and consult C coding resources such as The C Programming Language by Kernighan and Ritchie.

Additional Practice
~~~~~~~~~~~~~~~~~~~

  While not part of the exercise requirements, you can practice
  additionally by further extending the list application. All of these
  are likely to follow the same basic structure of 1 or more C functions
  being added to `list_funcs.c' and a corresponding command added to
  `list_main.c'. Note that you will also need to add new function
  prototypes to `list.h' for any new functions so that the compiler is
  aware of them.

  Some possible additions are.
  - `count' which returns the number of items currently in the list
  - `remove_index <N>' to remove a specific list item number;
    e.g. `remove_indx 4' to remove element 4 of the list
  - `drop_from <N>' which deletes all items starting at index `<N>' to
    the end of the list from the list
  - `drop_upto <N>' which deletes items from the beginning of the list
    up to index `<N>'
  - `load <file>' and `save <file>' which read a list from a file or
    store the current file in a list. These will require knowledge of
    C's basic I/O functions which we will study later but will required
    functionality for an upcoming project.

4 Command Echoing via -echo

The provided list_main program features an -echo option. When supplied, the program will "echo" commands that are entered by printing them back on the screen. Here is a brief shell session demonstrating the difference in behavior that running with -echo produces: 

# Running normally (no -echo option)
>> ./list_main
...
list> insert Apple
list> insert Banana
list> print
0: Apple
1: Banana
list> contains Banana
'Banana' is present
list> contains Orange
not found
list> exit

# Running with -echo (command echoing enabled)
>> ./list_main -echo
...
list> insert Apple
insert Apple                    # echoed
list> insert Banana
insert Banana                   # echoed
list> contains Banana
contains Banana                 # echoed
'Banana' is present
list> contains Orange
contains Orange                 # echoed
not found
list> exit
exit                            # echoed

At first the echoing feature seems pointless. Its strength comes from it use in "scripting" the program to allow commands to be read from a file rather than typed in directly. The -echo option is useful to produce output that resembles an interactive session and is easier to understand. Below is a session demonstrating the use of echoing with a script to produce good looking results. 

# show contents of a "script", list commands that are in a text file
>> file script.txt
script.txt: ASCII text

>> cat script.txt
insert Apple
insert Banana
contains Banana
contains Orange
exit

# run list_main reading commands from script.txt rather than typed input
# since input commands are not printed by default, the output looks crummy
>> ./list_main < script.txt
Linked List Demo
Commands:
  print:          shows the current contents of the list
  clear:          eliminates all elements from the list
  exit:           exit the program
  insert thing:   inserts the given string into the list
  get index:      get the item at the given index
  contains thing: determine if the given thing is in the list (IMPLEMENT)
  remove thing:   remove the given thing if it is present (IMPLEMENT)
list> list> list> 'Banana' is present
list> not found
list> >> 

# run list_main with input from script.txt with command echoing enabled
# repeating the commands makes the output look like an interactive session
>> ./list_main -echo < script.txt
Linked List Demo
Commands:
  print:          shows the current contents of the list
  clear:          eliminates all elements from the list
  exit:           exit the program
  insert thing:   inserts the given string into the list
  get index:      get the item at the given index
  contains thing: determine if the given thing is in the list (IMPLEMENT)
  remove thing:   remove the given thing if it is present (IMPLEMENT)
list> insert Apple
list> insert Banana
list> contains Banana
'Banana' is present
list> contains Orange
not found
list> exit
>>

Command echoing like this for scripted input is common for Read / Eval / Print Loops (REPLs) like shells, Python, R, Lua, Scheme, Racket, and many others. It is also extremely useful in testing as one can see in a test cast the input command along with the output and judge whether it is correct. This tactic is used in the testing program for the lab.

The above example also demonstrates I/O redirection using the shell syntax < script.txt. This capability will be discussed later in the course. It allows input that must normally be typed to come from elsewhere, such as a file, without any modifications to the program.

Echoing will be a required feature of a future project and students will be encouraged to examine this Lab again to replicate the technique demonstrated here.
